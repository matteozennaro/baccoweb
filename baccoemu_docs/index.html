Welcome to baccoemu’s documentation!¶

The bacco emulators are part of the bacco project. Visit us on our main website. Our emulators provide predictions for

  • the cold matter linear power spectrum

  • the cold matter nonlinear power spectrum

  • the baryonic boost to the cold matter nonlinear power spectrum

in a wide cosmological parameter space, including dynamical dark energy and massive neutrinos.

Our emulators are publicly available; please, follow the links above to be see te corresponding papers on the NASA/ADS website, where you can find all the references to credit our work.

Note

The bacco project is under constant development and new versions of the emulators become available as we improve them. Follow our public repository to make sure you are always up to date with our latest release.

Installation¶

As a quick summary, you shouldn’t need anything more than

pip install baccoemu [--user]

or, if you already installed baccoemu and you want to update to the latest version,

pip install --upgrade baccoemu [--user]

If you downloaded baccoemu from the public repository, you should enter the baccoemu folder end execute

pip install . [--user]

Warning

the bacco emulator only works in a Python 3 environment; the data file at its core cannot be unpickled by Python 2.x; in case your pip command doesn’t link to a Python 3 pip executable, please modify the line above accordingly (e.g. with pip3 instead of pip)

Note

The bacco emulator depends on some external packages, namely

  1. numpy

  2. sklearn

  3. GPy

  4. keras

  5. tensorflow

  6. matplotlib

  7. scipy

  8. camb

  9. progressbar2

The installation process will automatically try to install them if they are not already present.

Quickstart¶

All of the bacco emulators take a dictionary containing the desired cosmological parameters. This should look like

params = {
    'omega_matter'  :  0.315,
    'sigma8'        :  0.83,
    'omega_baryon'  :  0.05,
    'ns'            :  0.96,
    'hubble'        :  0.67,
    'neutrino_mass' :  0.0,
    'w0'            : -1.0,
    'wa'            :  0.0,
    'expfactor'     :  1
}

Please note that omega_matter and sigma8 refer to the density parameter and linear variance of cold matter (cdm + baryons), which does not correspond to the total matter content in massive neutrino cosmologies.

You can load the nonlinear matter power spectrum emulator and the linear matter power spectrum emulator via

import baccoemu
emulator = baccoemu.Matter_powerspectrum()

From this object, the user can obtain three quantities:

k, Q = emulator.get_nonlinear_boost(params)
k, pk = emulator.get_linear_pk(params, k=k)
k, pknl = emulator.get_nonlinear_pk(params, baryonic_boost=False)

These are the nonlinear boost Q (which, multiplied by the corresponding linear power spectrum gives the nonlinear power spectrum), the emulated linear power spectrum pk and the emulated nonlinear power spectrum pknl.

Note that to get the nonlinear power spectrum, baccoemu will internally multiply the boost factor Q by the emulated linear power spectrum. If you want to use another linear power spectrum, you can pass it via

k, pknl = emulator.get_nonlinear_pk(params, baryonic_boost=False, pk_lin=your_linear_pk)

When baryonic corrections are required, the parameter dictionary must include the relevant parameters, such as

params = {
    'omega_matter'  :  0.315,
    'sigma8'        :  0.83,
    'omega_baryon'  :  0.05,
    'ns'            :  0.96,
    'hubble'        :  0.67,
    'neutrino_mass' :  0.0,
    'w0'            : -1.0,
    'wa'            :  0.0,
    'expfactor'     :  1,

    'M_c'           :  14,
    'eta'           : -0.3,
    'beta'          : -0.22,
    'M1_z0_cen'     : 10.5,
    'theta_out'     : 0.25,
    'theta_inn'     : -0.86,
    'M_inn'         : 13.4
}

In this case, the baryonic boost is obtained through

k, S = emulator.get_baryonic_boost(params)

Finally, the nonlinear matter power spectrum with baryonic effects can be obtained with

k, pknl = emulator.get_nonlinear_pk(params, baryonic_boost=True)

Complete API¶

class baccoemu.matter_powerspectrum.Matter_powerspectrum(emu_type='nn', linear=True, nonlinear_boost=True, baryonic_boost=True, verbose=True)¶

A class to load and call the baccoemu for the matter powerspectrum. By default, the linear power spectrum (described in Aricò et al. 2021), the nonlinear boost (described in Angulo et al. 2020), and the baryonic boost (described in Aricò et al. 2020c) are loaded.

Parameters

  • emu_type (string, optional) – whether to load the Neural Network (‘nn’) or the Gaussian Process (‘gp’) emulator. The latter option is available only for the nonlinear boost emulator. Default, ‘nn’

  • linear (boolean, optional) – whether to load the linear emulator. Default, True.

  • nonlinear_boost (boolean, optional) – whether to load the nonlinear boost emulator. Default, True.

  • baryonic_boost (boolean, optional) – whether to load the baryonic boost emulator. Default, True.

  • verbose (boolean, optional) – whether to activate the verbose mode. Default True.

get_baryonic_boost(coordinates, k=None)¶

Evaluate the baryonic emulator at a set of coordinates in parameter space.

The coordinates must be specified as a dictionary with the following keywords:

  1. ‘omega_matter’

  2. ‘omega_baryon’

  3. ‘sigma8’

  4. ‘hubble’

  5. ‘ns’

  6. ‘Mnu’

  7. ‘w0’

  8. ‘wa’

  9. ‘expfactor’

  10. ‘M_c’

  11. ‘eta’

  12. ‘beta’

  13. ‘M1_z0_cen’

  14. ‘theta_out’

  15. ‘theta_inn’

  16. ‘M_inn’

Parameters

  • coordinates (dict) – a set of coordinates in parameter space

  • k (array_like, optional) – a vector of wavemodes in h/Mpc at which the nonlinear boost will be computed, if None the default wavemodes of the baryonic emulator will be used, defaults to None

Returns

k and S(k), the emulated baryonic boost of P(k)

Return type

tuple

get_nonlinear_pk(coordinates, k=None, k_lin=None, pk_lin=None, baryonic_boost=False)¶

Compute the prediction of the nonlinear cold matter power spectrum.

The coordinates must be specified as a dictionary with the following keywords:

  1. ‘omega_matter’

  2. ‘omega_baryon’

  3. ‘sigma8’

  4. ‘hubble’

  5. ‘ns’

  6. ‘Mnu’

  7. ‘w0’

  8. ‘wa’

  9. ‘expfactor’

If baryonic_boost=True, baryonic effects are included. In this case, the following keywords must be specified:

  1. ‘M_c’

  2. ‘eta’

  3. ‘beta’

  4. ‘M1_z0_cen’

  5. ‘theta_out’

  6. ‘theta_inn’

  7. ‘M_inn’

Parameters

  • coordinates (dict) – a set of coordinates in parameter space

  • k (array_like, optional) – a vector of wavemodes in h/Mpc at which the nonlinear boost will be computed, if None the default wavemodes of the emulator will be used, defaults to None

  • k_lin (array_like, optional) – a vector of wavemodes in h/Mpc, if None the wavemodes used by the linear emulator are returned, defaults to None

  • pk_lin (array_like, optional) – a vector of linear power spectrum computed at k_lin, if None the linear emulator will be called, defaults to None

Returns

k and the nonlinear P(k)

Return type

tuple

baccoemu.baryonic_boost.get_baryon_fractions(coordinates, M_200c)¶

Compute the mass fraction of the different baryonic components, following the baryonic correction model described in Aricò et al 2020b (see also Aricò et al 2020a).

The coordinates must be specified as a dictionary with the following keywords:

  1. ‘omega_matter’

  2. ‘omega_baryon’

  3. ‘sigma8’

  4. ‘hubble’

  5. ‘ns’

  6. ‘Mnu’

  7. ‘w0’

  8. ‘wa’

  9. ‘expfactor’

  10. ‘M_c’

  11. ‘eta’

  12. ‘beta’

  13. ‘M1_z0_cen’

  14. ‘theta_out’

  15. ‘theta_inn’

  16. ‘M_inn’

Parameters

  • coordinates (dict) – a set of coordinates in parameter space

  • M_200 – Halo mass inside a sphere which encompass a density 200 times larger than the critical density of the Universe.

:type array_like

Returns

a dictionary containing the baryonic components mass fractions, with the following keywords:

  1. ‘ej_gas’ -> ejected gas

  2. ‘cen_galaxy’ -> central galaxy

  3. ‘sat_galaxy’ -> satellite galaxy

  4. ‘bo_gas’ -> bound gas

  5. ‘re_gas’ -> reaccreted gas

  6. ‘dark_matter’ -> dark matter

  7. ‘gas’ -> total gas

  8. ‘stellar’ -> total stars

  9. ‘baryon’ -> total baryons

Return type

dict
